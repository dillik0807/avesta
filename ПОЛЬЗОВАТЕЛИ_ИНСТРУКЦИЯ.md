# 👤 Управление пользователями

## ✅ Что добавлено

Раздел "Пользователи" полностью интегрирован в систему с функционалом как в старом проекте.

## 📋 Функции

### 1. Добавление пользователя
- **Логин** - уникальное имя пользователя
- **Пароль** - минимум 4 символа
- **Роль**:
  - `Администратор` - полный доступ ко всем функциям
  - `Завсклада` - управление складом
  - `Кассир` - работа с расходами и погашениями
  - `Руководитель` - просмотр отчетов
- **Группа складов** - ограничение доступа к определенной группе складов (необязательно)

### 2. Редактирование пользователя
- Нажмите кнопку ✏️ рядом с пользователем
- Измените данные в форме
- Нажмите "Добавить пользователя" для сохранения
- Если не указываете новый пароль, старый пароль сохраняется

### 3. Удаление пользователя
- Нажмите кнопку 🗑️ рядом с пользователем
- Подтвердите удаление
- ⚠️ Пользователя `admin` удалить нельзя

### 4. Смена пароля
- Выберите пользователя из списка
- Введите текущий пароль
- Введите новый пароль (минимум 4 символа)
- Подтвердите новый пароль
- Нажмите "Изменить пароль"

**Права доступа:**
- Администратор может менять пароль любому пользователю
- Обычные пользователи могут менять только свой пароль

## 🔐 Безопасность

- Все пароли хешируются с помощью bcrypt перед сохранением в базу данных
- Пароли никогда не хранятся в открытом виде
- Используется JWT токен для аутентификации

## 🗄️ База данных

Таблица `users` уже существует в PostgreSQL со следующими полями:
- `id` - уникальный идентификатор
- `username` - логин пользователя
- `password_hash` - хешированный пароль
- `role` - роль пользователя
- `warehouse_group` - группа складов (может быть NULL)

## 🚀 Как использовать

1. Откройте систему в браузере: http://localhost:8080
2. Войдите как администратор (admin / admin123)
3. Перейдите в раздел "👤 Пользователи" в меню
4. Добавьте новых пользователей с нужными ролями
5. Настройте группы складов при необходимости

## 📝 Примеры

### Добавление завсклада
```
Логин: sklad1
Пароль: sklad123
Роль: Завсклада
Группа складов: Группа А
```

### Добавление кассира
```
Логин: kassir1
Пароль: kassir123
Роль: Кассир
Группа складов: (оставить пустым для доступа ко всем)
```

## ⚠️ Важно

- Первый пользователь `admin` создается автоматически при инициализации базы данных
- Пароль по умолчанию: `admin123`
- **Обязательно смените пароль администратора после первого входа!**

## 🔧 API Endpoints

Backend предоставляет следующие endpoints:

- `GET /api/users` - получить всех пользователей
- `POST /api/users` - добавить пользователя
- `PUT /api/users/:id` - обновить пользователя
- `DELETE /api/users/:id` - удалить пользователя
- `POST /api/users/change-password` - сменить пароль

Все endpoints требуют авторизации через JWT токен.
