⚠️ ВАЖНО: НУЖНО ВЫПОЛНИТЬ МИГРАЦИЮ БД

📋 ПРОБЛЕМА:
Бейджи статуса не работают, потому что в таблице users нет полей:
- require_password_change
- is_blocked

🔧 РЕШЕНИЕ:

Вариант 1: Автоматическая миграция (рекомендуется)
================================================

1. Убедитесь, что PostgreSQL запущен
2. Убедитесь, что сервер остановлен (Ctrl+C в терминале сервера)
3. Выполните команду:

   cd postgres-version
   node backend/scripts/add-user-status-fields.js

4. Запустите сервер снова:

   node backend/server.js


Вариант 2: Ручное добавление через SQL
=======================================

1. Откройте pgAdmin или psql
2. Подключитесь к базе данных warehouse_db
3. Выполните SQL команду:

```sql
ALTER TABLE users 
ADD COLUMN IF NOT EXISTS require_password_change BOOLEAN DEFAULT false,
ADD COLUMN IF NOT EXISTS is_blocked BOOLEAN DEFAULT false;
```

4. Проверьте, что поля добавлены:

```sql
SELECT column_name, data_type, column_default
FROM information_schema.columns
WHERE table_name = 'users'
ORDER BY ordinal_position;
```

Вы должны увидеть:
- require_password_change | boolean | false
- is_blocked | boolean | false


📊 ПОСЛЕ МИГРАЦИИ:

1. Перезапустите сервер (если он был запущен)
2. Обновите страницу в браузере (F5)
3. Перейдите в "Управление" → "Пользователи"
4. Теперь бейджи статуса должны работать:
   - ✅ Активен (зеленый) - по умолчанию для всех
   - 🔑 Смена пароля (желтый) - если галочка установлена
   - 🔒 Заблокирован (красный) - если галочка установлена


🧪 ТЕСТИРОВАНИЕ:

1. Создайте нового пользователя
2. Установите галочку "Требуется смена пароля при входе"
3. Сохраните
4. В таблице должен появиться бейдж: 🔑 Смена пароля

5. Отредактируйте пользователя
6. Установите галочку "Заблокировать пользователя"
7. Сохраните
8. В таблице должны быть оба бейджа: 🔒 Заблокирован 🔑 Смена пароля


❓ ЕСЛИ НЕ РАБОТАЕТ:

1. Проверьте консоль браузера (F12) на ошибки
2. Проверьте, что поля добавлены в БД:
   
   SELECT * FROM users LIMIT 1;
   
   Должны быть колонки require_password_change и is_blocked

3. Проверьте, что сервер перезапущен после миграции

4. Очистите кэш браузера (Ctrl+Shift+Delete)


✅ ГОТОВО!
После выполнения миграции все бейджи статуса будут работать корректно.
