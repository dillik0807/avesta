✅ ИСПРАВЛЕНО УДАЛЕНИЕ В УПРАВЛЕНИИ!
=====================================

🔧 Проблема:
Ошибка 500 при попытке удалить записи, которые используются 
в других таблицах (foreign key constraint violation).

✅ Решение:
Добавлена проверка использования перед удалением:

1. ФИРМЫ:
   - Проверяется использование в приходе и расходе
   - Если используется - показывается сообщение с количеством записей
   - Если не используется - удаляется

2. СКЛАДЫ:
   - Проверяется использование в приходе и расходе
   - Если используется - показывается сообщение
   - Если не используется - удаляется

3. ТОВАРЫ:
   - Проверяется использование в приходе и расходе
   - Если используется - показывается сообщение
   - Если не используется - удаляется

4. КЛИЕНТЫ:
   - Проверяется использование в расходе и погашениях
   - Если используется - показывается сообщение
   - Если не используется - удаляется

📋 ПРИМЕР СООБЩЕНИЯ:
"Невозможно удалить склад. Он используется в 5 записях 
прихода и 3 записях расхода."

✅ ТЕПЕРЬ:
- Нельзя удалить используемые записи
- Показывается понятное сообщение об ошибке
- Можно удалить только неиспользуемые записи
- Редактирование работает всегда

🚀 Backend сервер перезапущен на порту 3001

Попробуйте снова удалить записи - теперь будет показываться 
информативное сообщение вместо ошибки 500!
