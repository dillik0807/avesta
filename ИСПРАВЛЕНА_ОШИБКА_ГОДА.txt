ПРОБЛЕМА:
---------
При добавлении прихода/расхода появлялась ошибка:

❌ Error: Ошибка добавления прихода: insert or update on 
   table "income" violates foreign key constraint "income_year_fkey"

ПРИЧИНА:
--------
Пользователь переключился на год 2026, но в таблице years 
нет записи для 2026 года. PostgreSQL требует чтобы год 
существовал в таблице years перед добавлением данных.

РЕШЕНИЕ:
--------
Добавлена автоматическая проверка и создание года в базе 
данных перед добавлением записи.

КОД ИЗМЕНЕНИЯ:
--------------

Добавлено в каждый роут перед вставкой данных:

```javascript
// Проверка и создание года если не существует
const yearCheck = await client.query(
    'SELECT id FROM years WHERE year = $1', 
    [year]
);

if (yearCheck.rows.length === 0) {
    console.log(`📅 Создание года ${year} в базе данных`);
    await client.query(
        'INSERT INTO years (year, last_modified) VALUES ($1, $2)',
        [year, Date.now()]
    );
}
```
