## 🚀 Запуск за 3 шага

### Windows:
```cmd
1. Откройте папку postgres-version
2. Дважды кликните START.bat
3. Откройте frontend/index.html в браузере
```

### Linux/Mac:
```bash
chmod +x START.sh
./START.sh
```

### Вручную:
```bash
cd postgres-version/backend
npm install
npm run test-connection
npm run init-db
npm start
```

## 📋 Что делает каждая команда

### `npm run test-connection`
Проверяет подключение к PostgreSQL на Railway
- ✅ Подключение к базе
- ✅ Версия PostgreSQL
- ✅ Список таблиц
- ✅ Список пользователей

### `npm run init-db`
Создает структуру базы данных
- Создает все таблицы
- Создает администратора (admin/admin123)
- Добавляет начальные данные

### `npm start`
Запускает сервер на http://localhost:3000

## 📊 API Endpoints

После запуска доступны:

**Аутентификация:**
- `POST /api/auth/login` - Вход в систему

**Данные:**
- `GET /api/data/:year` - Получить данные за год
- `POST /api/data/sync` - Синхронизация

**Приход:**
- `GET /api/income` - Список прихода
- `POST /api/income` - Добавить приход
- `PUT /api/income/:id` - Обновить
- `DELETE /api/income/:id` - Удалить

**Расход:**
- `GET /api/expense` - Список расхода
- `POST /api/expense` - Добавить расход
- `PUT /api/expense/:id` - Обновить
- `DELETE /api/expense/:id` - Удалить

**Погашения:**
- `GET /api/payments` - Список погашений
- `POST /api/payments` - Добавить погашение
- `PUT /api/payments/:id` - Обновить
- `DELETE /api/payments/:id` - Удалить

**Справочники:**
- `GET /api/management/dictionaries` - Все справочники
- `POST /api/management/companies` - Добавить фирму
- `POST /api/management/warehouses` - Добавить склад
- `POST /api/management/products` - Добавить товар
- `POST /api/management/clients` - Добавить клиента

## 🔄 Миграция данных из Firebase

Если у вас есть данные в старой Firebase версии:

1. Экспортируйте данные из Firebase Console (Export JSON)
2. Сохраните как `firebase-export.json` в корне проекта
3. Запустите:
```bash
cd backend
node scripts/migrate-from-firebase.js
```

## ❓ Troubleshooting

### Ошибка подключения к БД
```bash
# Проверьте DATABASE_URL
cat backend/.env | grep DATABASE_URL

# Или на Windows
type backend\.env | findstr DATABASE_URL
```

### Порт 3000 занят
Измените PORT в `backend/.env`:
```
PORT=3001
```

### Таблицы не созданы
```bash
cd backend
npm run init-db
```

### CORS ошибки
Добавьте URL фронтенда в `CORS_ORIGINS` в `.env`
