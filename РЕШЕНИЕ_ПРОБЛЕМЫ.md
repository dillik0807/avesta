# ⚠️ Ошибка: Failed to fetch

## Причина

Ошибка "Failed to fetch" означает, что **сервер не запущен**.

Фронтенд пытается подключиться к `http://localhost:3000`, но сервер не отвечает.

---

## ✅ Решение (3 шага)

### Шаг 1: Откройте командную строку

**Windows:**
- Нажмите `Win + R`
- Введите `cmd`
- Нажмите Enter

**Или:**
- Откройте PowerShell
- Или Git Bash

### Шаг 2: Перейдите в папку backend

```bash
cd путь\к\проекту\postgres-version\backend
```

Например:
```bash
cd C:\Users\YourName\Desktop\postgres-version\backend
```

### Шаг 3: Запустите сервер

**Первый раз (установка и инициализация):**
```bash
npm install
npm run init-db
npm start
```

**Последующие разы:**
```bash
npm start
```

---

## 🚀 Быстрый способ (Windows)

1. Откройте папку `postgres-version`
2. Дважды кликните на `START.bat`
3. Дождитесь запуска сервера

## 🔧 Troubleshooting

### Проблема: "Порт 3000 уже используется"

**Решение 1:** Закройте другое приложение на порту 3000

**Решение 2:** Измените порт
1. Откройте `backend/.env`
2. Измените `PORT=3000` на `PORT=3001`
3. Перезапустите сервер

### Проблема: "Cannot find module"

```bash
cd backend
npm install
```

### Проблема: "Database connection error"

```bash
cd backend
npm run test-connection
```

Если ошибка - проверьте `DATABASE_URL` в `backend/.env`

### Проблема: "Table does not exist"

```bash
cd backend
npm run init-db
```
